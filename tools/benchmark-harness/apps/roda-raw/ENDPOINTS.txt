3. POST /json/large          - Large JSON payload (no validation)
4. POST /json/very-large     - Very large JSON payload (no validation)

-------------------------------

VALIDATION STATUS:
-----------------
All validation REMOVED. Server accepts any JSON input and echoes back.
No Dry::Schema dependency. Pure Roda performance measurement.

PLUGINS USED:
------------
- json         : Automatic JSON response serialization
- json_parser  : Automatic JSON request body parsing
- all_verbs    : Support for all HTTP methods
- type_routing : Type-based path parameter matching

--------------------
- Port: ARGV[0] or 8000
- Host: 0.0.0.0
- Threads: 1:1 (single-threaded for benchmarking)
- Server: Puma
- Silent mode: true
